# CLAUDE.md 특화 점검

대상이 CLAUDE.md(또는 변형 — `CLAUDE.local.md`, `~/.claude/CLAUDE.md`, monorepo 서브디렉토리 CLAUDE.md)일 때 SKILL.md 범용 점검에 *더해* 적용한다.

## 추가 삭제 신호

- 범용 best practice·일반론(이 프로젝트 특정이 아닌 조언) → **삭제**. CLAUDE.md는 *이 프로젝트에서만 참인 것*만 담는다.
- 여러 CLAUDE.md(루트·서브디렉토리·`~/.claude`)에 같은 내용 중복 → 가장 가까운 한 곳만 두고 **pointer**.
- 일회성 이력(해결된 버그·특정 커밋·날짜)이 CLAUDE.md에 특히 자주 낀다 → **삭제** (범용 신호 재확인).

## 누락 플래그 — 플래그만, 추가는 사용자 확인

아래 핵심이 없으면 *플래그*한다. 무엇을 채울지는 사용자가 정한다 — 이 스킬은 내용을 창작하지 않는다.

- **자주 쓰는 명령** — build·test·lint·run 등 (매번 탐색하게 만드는 누락)
- **아키텍처 개요** — 주요 모듈·디렉토리 역할 한눈 지도
- **gotcha·함정** — 코드만 봐선 모르는 비자명 제약·관례
- **테스트 방법** — 어떻게 돌리고 무엇을 PASS로 보는지
- **코드 컨벤션** — 프로젝트 고유 네이밍·패턴 (범용 스타일 말고)

## stale 플래그 — 플래그만, 삭제는 사용자 확인

- 존재하지 않는 경로·파일·앵커를 가리키는 참조
- 실패할 명령(틀린 경로·삭제된 스크립트)
- 더는 맞지 않는 디렉토리 구조 서술

stale 판정은 코드베이스 cross-check 영역이라 형식 다이어트 범위를 넘는다 — 의심을 *플래그*하고 실제 수정은 사용자 확인.

## 위치별 성격 — audit 잣대가 달라진다

- **프로젝트 루트 `CLAUDE.md`** — 팀 공유·커밋됨. 위 누락 플래그·삭제 신호를 정상 적용.
- **`CLAUDE.local.md`** — gitignore·개인·임시(진행 중 사이클 한정). 누락 플래그 잣대 완화 — 임시 메모라 완결성을 요구하지 않는다.

- **monorepo 서브디렉토리 `CLAUDE.md`** — 그 하위 트리 한정. 루트와 중복되는 내용은 삭제 신호(루트로 pointer).
