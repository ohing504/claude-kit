# iPhone Notes Digest

맥/아이폰 Apple Notes에 쌓인 메모를 **폴더 선택 → 수집(추출+영상 해석) → 한 장의 다이제스트(사실) 문서로 보고**한다. 메모앱은 캡처 입구일 뿐 보관처가 아니라는 전제 — 사용자가 한 번에 훑고 처분(자산화/폐기)할 수 있게, "이게 뭐였는지"가 사실로 적힌 검토 가능한 상태로 만든다.

## 이 스킬이 하는 일 / 안 하는 일 (책임 경계)

- **한다**: 메모 본문 추출, 메모 안 링크·미디어를 실제로 fetch해 내용 해석(영상은 음성까지), 메모별 섹션으로 정리한 **다이제스트(사실) 문서 1장** 생성.
- **안 한다 — 판단**: 처분 판단(자산화/폐기/보류), 어디로 흡수할지(라우팅), 무엇을 삭제할지 *결정*, 과거 결정 학습. 이건 전부 스킬 밖이다. 스킬은 **사실만 기록하고 판단하지 않는다.** 그 문서를 받은 사용자(또는 사용자의 노트 시스템 — 예: 개인 Obsidian/Notion 등 PKM)가 자기 기준으로 흡수·폐기를 정한다.
- **삭제는 메커니즘만**: 리뷰 흐름은 메모앱 원본을 **자동으로** 삭제하지 않는다. 사용자(또는 그 노트 시스템)가 "무엇을 지울지" 정한 뒤 "처리한 메모 지워줘"라고 토큰을 넘기면, 삭제 *실행*만 `dispose_notes.sh`(가드+DRY RUN 내장)로 한다 — 메모앱 컨트롤은 이 스킬 영역이라 raw osascript를 즉석에서 짜지 않는다. "무엇을 지울지"는 스킬이 정하지 않는다.

## 데이터 ≠ 로직 경계

이 스킬 본문은 **로직(어떻게 추출·해석·정리하는가)만** 담는다. 추출된 실제 메모 내용·링크 해석 결과는 사용자가 지정한 출력 경로(없으면 현재 작업 디렉토리)에만 쓰고, 이 스킬이 설치된 레포에는 절대 쓰지 않는다. 임시 수집물·오디오는 `WORK_DIR`(기본 `/tmp/notes-digest-work`, gitignore 대상)에만 두고 작업 후 지운다.

## 필요 도구

직접 설치해야 하는 것은 두 가지뿐. 나머지는 `uvx`가 실행 시 자동으로 가져온다.

- **uv** — `uvx`로 yt-dlp·STT 엔진을 격리 실행. (`brew install uv` 또는 `curl -LsSf https://astral.sh/uv/install.sh | sh`)
- **ffmpeg** — 오디오·프레임 추출용 시스템 바이너리. (`brew install ffmpeg`)

`collect.sh` 첫 실행 시 두 도구의 존재를 확인하고 없으면 안내한다. yt-dlp·STT 엔진은 별도 설치 없이 각 스크립트가 `uvx`로 호출한다.

## 워크플로우

### 1. 폴더 선택 — *반드시 먼저 묻는다*

`$ARGUMENTS`가 있고 **짧은 단어·구** (폴더명으로 해석되는 것)면 그 이름을 폴더로 바로 사용하고 이 단계를 건너뛴다(단, account 중복이 있으면 확인). 자연어 문장이거나 동사가 포함된 설명이면 폴더명이 아닌 맥락 정보로 취급하고 일반 흐름(목록 제시 → 선택)으로 진행한다. **없으면** `scripts/list_folders.sh`로 폴더 목록과 각 폴더 메모 수를 보여주고, **사용자에게 어떤 폴더를 처리할지 먼저 묻는다.** 이게 0순위다.

```bash
scripts/list_folders.sh
# account별로 묶여 나오고, 잠긴 메모가 있으면 그 수도 함께 표기된다:
#   [iCloud]
#   레시피 (12) / 여행 (8, 잠김 2) / 보관 (5) ...
#   [Google]
#   Notes (0)
```

묻지 않고 전체를 도는 것은 금물 — 메모앱엔 **잠긴 폴더·옛 개인글·민감정보(주민번호·비밀번호 등)**가 섞여 있을 수 있다. 사용자가 명시적으로 "전체"라고 할 때만 전 폴더를 돈다.

- **잠긴 개별 메모**는 목록에 `(N, 잠김 M)`으로 수가 표기된다. 본문은 osascript로 안 읽혀 추출 시 `[LOCKED]`로 남으니, 그 폴더를 골랐다면 "M건은 잠겨서 본문 못 봄"을 사용자에게 알린다.
- **account별 그룹**(`[iCloud]` / `[Google]` 등)이 머리에 붙는다. 같은 이름 폴더가 account마다 따로 있을 수 있으니(예: 양쪽 "Notes") 어느 account인지 확인해 처리한다. 이메일 계정(Gmail/Exchange) 폴더도 여기 함께 나온다.

### 2. 수집 — `collect.sh`

선택된 폴더로 추출 + 영상 해석을 한 번에 돌린다:

```bash
scripts/collect.sh "레시피" "여행"
# 결과: $WORK_DIR/raw.txt (메모 원본), $WORK_DIR/enrich/NNN.txt (영상별 해석)
```

지정 폴더의 메모를 추출하고, 그 안 소셜 미디어 URL(인스타·유튜브·틱톡·**Threads**)을 전부 enrich(캡션 + 캡션 얕으면 STT 자동)한다. Threads는 텍스트 게시물이면 본문이 CAPTION으로, 영상이면 기존 릴스처럼 동작한다. 인스타 rate limit 회피로 URL 사이에 쉼이 있어, 많으면 몇 분 걸린다.

> **이 스킬의 핵심 판단(여기서 자동으로 일어남):** 캡션이 짧거나(한 줄·해시태그만·"댓글에 ○○ 남겨주세요"류) 비어 있으면, 정보가 없는 게 아니라 **음성에 있다.** `collect.sh`는 캡션이 얕은 영상에 자동으로 STT를 돌린다. **그러므로 캡션이 얕다고 "내용 없음"으로 적지 말고, STT 결과를 보고 사실을 적는다.**

캡션 "충분" 기준(기본 50자)·언어는 `enrich_video.sh` 인자로 조정 가능하다. 유튜브는 자막이 풍부하니 STT보다 자막이 싸지만(`yt-dlp --write-auto-subs`), 인스타는 자동자막을 거의 안 줘서 캡션→STT 직행이 실용적이다.

### 3. 해석 — 다이제스트 작성 (사실만)

`$WORK_DIR/raw.txt`(메모 원본)와 `$WORK_DIR/enrich/`(영상 해석)를 읽어 메모별 섹션을 작성한다. 사용자 지정 경로(없으면 `./notes-digest-<날짜>.md`)에 쓴다. **이 문서는 "무엇인지(사실)"만 적는다 — 살릴지/버릴지는 적지 않는다.** 그건 문서를 받는 사용자(시스템)의 몫이다.

- **각 메모의 `id:`(x-coredata)를 섹션 ID로 그대로 옮긴다** — 소비자가 흡수 매칭·삭제 토큰으로 쓰는 안정 핸들이다(제목·생성시각은 동명·동시각에 깨진다).
- **본문이 `[LOCKED]`인 메모**는 "잠김 — 본문 미열람"으로 섹션에 남기고(내용 지어내지 않는다), 상단 집계에 잠긴 건수를 포함한다. `collect.sh`가 "잠겨서 본문 추출 못 한 메모: N"을 보고하니 그 수를 쓴다.
- 메모 본문의 영상 외 **웹 링크**도 여기서 처리한다. 의미 있는 외부 페이지는 WebFetch로 무엇에 관한 건지 한 줄 뽑고, 사내망·로컬(`192.168.*`)·죽은 옛 링크는 fetch 없이 "오래된/내부 링크"로 표기한다(가능하면 실제 도달 여부를 확인해 "도달 불가(404)" 같은 **사실**로 적는다 — 폐기 판단은 안 하고 사실만).
- 메모가 많으면(수십 건+) 이 읽기·요약은 **sub-agent에 위임**해 요약만 받는다 — main 컨텍스트를 아낀다.
- **시각이 핵심인 영상**(촬영·플레이팅·메이크업처럼 화면을 가리키며 설명 — 캡션·STT만으론 "무엇인지"가 안 잡히는 것)은 `scripts/extract_frames.sh <URL> [시작초] [끝초]`로 장면 전환 프레임을 뽑는다. 음성 타임스탬프가 아니라 *화면 전환* 기준이라 빠르게 바뀌는 예시를 놓치지 않는다(STT 시점만 뽑으면 한 구간에 여러 예시가 스쳐도 한 장만 건진다). 나온 `contact.jpg`(타일 시트, 자막이 박혀 어느 대목인지 보임)를 보고 **대표 컷만** 골라 다이제스트에 포함한다 — 모든 컷을 넣지 않는다. STT로 특정 대목 구간(예: 한 각도 설명 4~20초)을 알면 시작/끝초로 좁힌다.
  - 이때도 경계는 같다: 프레임은 "이 영상이 뭔지" 보여주는 **사실(해석)**이다. 그 컷을 어디로 자산화할지(메모 커버·Wiki 등)는 스킬이 아니라 사용자/노트 시스템 몫.
- **인스타 이미지 게시물·캐러셀**(슬라이드 안 텍스트가 본문 — 프롬프트·팁 모음 등): enrich가 `IMAGE_POST` 신호를 남기거나 `/p/` 링크의 캡션이 비면, `scripts/extract_carousel.sh <URL>`로 전 슬라이드를 받는다. **로그인 불필요** — `/embed/captioned/` 엔드포인트가 공개 게시물의 전 슬라이드 + 캡션을 비로그인으로 준다(yt-dlp/gallery-dl이 로그인 벽에 막혀도 이 경로는 뚫린다). 받은 `slide_NN.jpg`를 **다이제스트 작성 LLM이 비전으로 읽어** 본문(슬라이드별 텍스트)을 뽑는다. 영상의 extract_frames와 같은 경계: 추출=스크립트, 판독=LLM. 비공개·로그인 전용 게시물은 `NO_GQL_DATA`로 정직하게 표기.

메모별 섹션 템플릿:

```markdown
## 1. <메모 제목 또는 영상 한 줄>
- **생성**: 2026-03-14 · **폴더**: 레시피 · **ID**: `x-coredata://…/ICNote/p195` (raw.txt의 `id:` 그대로 — 흡수·삭제 토큰 키)
- **원문**: (메모에 사용자가 적은 코멘트, 있으면)
- **링크 해석**:
  - 📷 [인스타] @업로더 — "캡션/음성에서 뽑은 핵심" (음성 STT면 그렇게 표기)
  - 🎬 [유튜브] "영상 제목" — 채널명 (12:34)
  - 🔗 [웹] example.com — "페이지가 무엇에 관한 것인지 한 줄"
```

문서 맨 위에 다이제스트임을 밝히는 안내를 얹는다:

```markdown
# Notes Digest — <생성일자 기준 N건>

> 아래는 각 메모가 "무엇이었는지"를 정리한 사실 다이제스트입니다. 살릴지/버릴지(흡수·폐기)는 직접 정하세요.

(... 메모별 섹션 ...)

---
총 45건 (추출 불가 잠긴 메모 3건 별도 — 본문 없음)
```

추출하지 못한 메모(잠김·이메일 계정 등)가 있으면 그 개수를 문서 상단/하단에 정직하게 적는다 — "전부 봤다"는 착시를 만들지 않기 위함이다(사용자가 이 문서를 보고 흡수·삭제를 정하므로 누락 사실이 중요하다).

### 4. 마무리 — Wrap up

- 다이제스트 문서 경로를 사용자에게 알린다. **흡수·처분은 사용자(또는 그 노트 시스템)가 이 문서를 보고 결정한다.**
- `$WORK_DIR`의 임시 수집물·오디오를 지운다.
- 메모앱 원본은 기본적으로 건드리지 않는다. 사용자(시스템)가 처분을 정하고 "처리한 메모 지워줘"라고 토큰을 넘기면, `dispose_notes.sh <토큰>`로 먼저 DRY RUN(대상 확인) → `--confirm`으로 삭제한다. 토큰은 처리 끝난 메모/링크의 고유 ID. "무엇을 지울지"는 스킬이 정하지 않는다 — 넘겨받은 토큰만 실행한다.

## 스크립트

- **`list_folders.sh`** — account별 폴더 목록 + 메모 수(+잠긴 메모 수). *폴더 선택을 먼저 묻기* 위한 도구.
- **`collect.sh "폴더"...`** — 지정 폴더 추출 + 영상 일괄 enrich(캡션+STT). 수집의 전 과정.
- `extract_notes.sh ["폴더"]` — 메모 추출만 (collect.sh가 호출하는 하위 도구).
- `enrich_video.sh <URL> [캡션최소길이=50] [언어=ko]` — 영상 1개 캡션+필요시 STT (collect.sh 하위).
- `extract_frames.sh <URL> [시작초] [끝초]` — 시각 핵심 영상의 *장면 전환* 프레임 추출 + 컨택트 시트. 작성 LLM이 시트를 보고 대표 컷을 선별하는 용도(추출=스크립트, 선별=LLM). 구간을 주면 그 안에서만.
- `extract_carousel.sh <URL>` — 인스타 이미지 게시물/캐러셀의 전 슬라이드를 로그인 없이 추출(/embed/captioned/ → gql_data 파싱). 슬라이드 텍스트 판독은 작성 LLM(비전). 영상=extract_frames, 이미지 게시물=extract_carousel.
- `_env.sh` — 플랫폼별 STT 명령어 선택(`CK_WHISPER_CMD`). Apple Silicon이면 `uvx mlx-whisper`, 그 외 `uvx --from whisper-ctranslate2 whisper-ctranslate2`. enrich_video.sh가 source.
- `dispose_notes.sh [--confirm] <토큰>...` — 처분이 끝난 메모를 메모앱에서 삭제하는 **메커니즘**. 기본 DRY RUN, `--confirm`으로 실삭제. "무엇을 지울지"는 사용자(시스템)가 토큰으로 넘기고, 스킬은 실행만 한다.

## 안전

- 링크 해석은 공개 메타데이터·음성 조회 수준 — 인증·로그인이 필요한 콘텐츠를 우회하지 않는다(인스타가 막히면 막힌 대로 표기).
- 해석 결과를 지어내지 않는다 — fetch·STT 실패는 실패로 정직하게 표기한다.
