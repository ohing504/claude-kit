# iPhone Notes Review

맥/아이폰 Apple Notes에 쌓인 메모를 **폴더 선택 → 수집(추출+영상 해석) → 한 장의 리뷰 문서로 보고**한다. 메모앱은 캡처 입구일 뿐 보관처가 아니라는 전제 — 사용자가 한 번에 훑고 처분(자산화/폐기)할 수 있게 검토 가능한 상태로 만든다.

## 이 스킬이 하는 일 / 안 하는 일

- **한다**: 메모 본문 추출, 메모 안 링크·미디어를 실제로 fetch해 내용 해석(영상은 음성까지), 메모별 섹션으로 정리한 리뷰 문서 1장 생성, 각 메모에 추천 처분 *의견* 제시.
- **안 한다**: 처분 자체(자산화/폐기/어디로 보낼지)는 사용자가 리뷰 문서를 보고 결정한다. 메모앱 원본을 삭제하지 않는다.

처분과 목적지를 스킬이 정하지 않는 이유: "어디로 보낼지"는 사람마다 다른 개인 설정(누구는 Obsidian, 누구는 Notion, 누구는 그냥 폴더)이라 스킬에 박으면 그 사람 전용이 돼버린다. 스킬은 *판단 가능한 문서*까지만 만들고, 처분은 그 문서를 본 사용자에게 남긴다.

## 데이터 ≠ 로직 경계

이 스킬 본문은 **로직(어떻게 추출·해석·정리하는가)만** 담는다. 추출된 실제 메모 내용·링크 해석 결과는 사용자가 지정한 출력 경로(없으면 현재 작업 디렉토리)에만 쓰고, 이 스킬이 설치된 레포에는 절대 쓰지 않는다. 임시 수집물·오디오는 `WORK_DIR`(기본 `/tmp/notes-review-work`, gitignore 대상)에만 두고 작업 후 지운다.

## 필요 도구

- **yt-dlp** — 영상(인스타 릴스·유튜브·틱톡)의 캡션·메타·자막·오디오 추출. (`brew install yt-dlp`)
- **ffmpeg** — yt-dlp가 오디오를 변환할 때 사용. (`brew install ffmpeg`)
- **STT 엔진** (영상 음성 → 텍스트, *캡션 얕은 영상 처리 시에만* 필요):
  - 맥(Apple Silicon): **mlx-whisper** 가 가장 빠름(릴스당 ~6초, 모델 캐시 후). `pip install mlx-whisper`
  - 그 외 플랫폼: **faster-whisper** (크로스플랫폼).
  - 모델은 `whisper-large-v3-turbo` — 작고 빠른데 한국어 정확도가 충분하다(고유명사·전문용어까지).

도구가 없으면 임의로 설치하지 말고 위 명령을 안내한다. STT는 무거운 단계이니, 캡션으로 충분한 영상에는 쓰지 않는다(`collect.sh`가 캡션 길이로 자동 판단).

## 워크플로우

### 1. 폴더 선택 — *반드시 먼저 묻는다*

`scripts/list_folders.sh`로 폴더 목록과 각 폴더 메모 수를 보여주고, **사용자에게 어떤 폴더를 처리할지 먼저 묻는다.** 이게 0순위다.

```bash
scripts/list_folders.sh
# 예: Notes (37) / 레시피 (12) / 여행 (8) / 보관 (5) ...
```

묻지 않고 전체를 도는 것은 금물 — 메모앱엔 **잠긴 폴더·옛 개인글·민감정보(주민번호·비밀번호 등)**가 섞여 있을 수 있다. 사용자가 명시적으로 "전체"라고 할 때만 전 폴더를 돈다.

- **잠긴 폴더**(보안 폴더 등)는 목록·개수엔 나오지만 본문은 osascript로 안 읽힌다 — 골라도 추출이 비니, 사용자에게 알린다.
- **같은 이름 폴더가 둘** 보일 수 있다(계정별, 예: iCloud "Notes" / On My Mac "Notes"). 모호하면 사용자에게 어느 쪽인지 확인한다.

### 2. 수집 — `collect.sh`

선택된 폴더로 추출 + 영상 해석을 한 번에 돌린다:

```bash
scripts/collect.sh "레시피" "여행"
# 결과: $WORK_DIR/raw.txt (메모 원본), $WORK_DIR/enrich/NNN.txt (영상별 해석)
```

지정 폴더의 메모를 추출하고, 그 안 영상 URL을 전부 enrich(캡션 + 캡션 얕으면 STT 자동)한다. 인스타 rate limit 회피로 영상 사이에 쉼이 있어, 영상이 많으면 몇 분 걸린다.

> **이 스킬의 핵심 판단(여기서 자동으로 일어남):** 캡션이 짧거나(한 줄·해시태그만·"댓글에 ○○ 남겨주세요"류) 비어 있으면, 정보가 없는 게 아니라 **음성에 있다.** 검증 사례 — 세무 릴스는 캡션이 "댓글에 '온라인' 남겨주세요"뿐이었지만 음성엔 온라인 사업 세무 전략 전체가, 푸드 릴스는 캡션이 "이 각도면 맛있게 찍힙니다🍔"였지만 음성엔 촬영 각도 3가지 상세가 있었다. `collect.sh`는 캡션이 얕은 영상에 자동으로 STT를 돌린다. **그러므로 캡션이 얕다고 폐기 후보로 분류하지 말고, STT 결과를 보고 판단한다.**

캡션 "충분" 기준(기본 50자)·언어는 `enrich_video.sh` 인자로 조정 가능하다. 유튜브는 자막이 풍부하니 STT보다 자막이 싸지만(`yt-dlp --write-auto-subs`), 인스타는 자동자막을 거의 안 줘서 캡션→STT 직행이 실용적이다.

리뷰 작성은 **해석(사실)과 판정(처분)을 한 패스에 섞지 않고 두 패스로 나눈다.** 한 호흡에 쓰면 "버릴 거라 미리 정해두면 요약을 부실하게 쓰는" 식으로 해석이 결론에 끌려간다. 사실을 먼저 굳힌 뒤 그것만 보고 판정해야, 이 스킬이 가장 자주 틀리던 지점 — **캡션이 얕았던 영상을 STT 요약을 보고서야 판단** — 이 지켜진다.

### 3. 해석 — Interpret (사실만, 처분 의견 없이)

`$WORK_DIR/raw.txt`(메모 원본)와 `$WORK_DIR/enrich/`(영상 해석)를 읽어 메모별 섹션을 작성한다. 사용자 지정 경로(없으면 `./notes-review-<날짜>.md`)에 쓴다. **이 패스에서는 "무엇인지"만 쓴다 — 살릴지/버릴지는 적지 않는다.**

- 메모 본문의 영상 외 **웹 링크**도 여기서 처리한다. 의미 있는 외부 페이지는 WebFetch로 무엇에 관한 건지 한 줄 뽑고, 사내망·로컬(`192.168.*`)·죽은 옛 링크는 fetch 없이 "오래된/내부 링크"로 표기한다.
- 메모가 많으면(수십 건+) 이 읽기·요약은 **sub-agent에 위임**해 요약만 받는다 — main 컨텍스트를 아끼고, 해석이 처분 결론에 끌려가지 않게 격리하는 효과도 같이 본다.

메모별 섹션 템플릿(처분 줄은 4단계에서 채운다):

```markdown
## 1. <메모 제목 또는 영상 한 줄>
- **생성**: 2026-03-14 · **폴더**: 레시피
- **원문**: (메모에 사용자가 적은 코멘트, 있으면)
- **링크 해석**:
  - 📷 [인스타] @업로더 — "캡션/음성에서 뽑은 핵심" (음성 STT면 그렇게 표기)
  - 🎬 [유튜브] "영상 제목" — 채널명 (12:34)
  - 🔗 [웹] example.com — "페이지가 무엇에 관한 것인지 한 줄"
```

### 4. 판정 — Classify (해석본만 보고 처분 추천)

3단계에서 쓴 해석 섹션을 입력으로, 각 메모에 처분 추천 한 줄을 **별도 패스로** 붙인다. 메모 원본·enrich를 다시 보지 말고 *해석본 텍스트만* 근거로 판정한다 — 해석이 부실하면 그건 3단계로 돌아가 보완할 신호다.

각 메모 섹션 끝에 추가:

```markdown
- **추천 처분**: 💎 자산화 후보 (이유) / 🗑 폐기 후보 (이유) / ⏸ 보류 (이유)
```

추천 처분 기준(의견 제시용, 강제 아님):
- **자산화 후보**: 살아있는 아이디어·계속 참고할 링크·할 일 씨앗·**음성에 알맹이가 있던 영상**.
- **폐기 후보**: 죽은 링크·이미 처리된 것·맥락 없는 한 줄 잔재. (단 영상은 해석본의 STT 요약을 확인한 뒤에만 — 캡션 얕다고 버리지 않는다.)
- **보류**: 지금 판단 어려운 것.

판정을 마치면 문서 맨 위에 안내와 한 줄 요약을 얹는다:

```markdown
# Notes Review — <생성일자 기준 N건>

> 처분은 직접 결정하세요. 아래 "추천"은 의견일 뿐입니다.

(... 메모별 섹션 ...)

---
총 45건 — 자산화 후보 11, 폐기 후보 24, 보류 10
```

### 5. 마무리 — Wrap up

- 리뷰 문서 경로를 사용자에게 알린다. **처분은 사용자가 이 문서를 보고 결정한다.**
- `$WORK_DIR`의 임시 수집물·오디오를 지운다.
- 메모앱 원본은 건드리지 않는다. 사용자가 "처리한 메모 지워줘"라고 명시할 때만, 그것도 확인을 거쳐 처리한다.

## 스크립트

- **`list_folders.sh`** — 폴더 목록 + 메모 수. *폴더 선택을 먼저 묻기* 위한 도구.
- **`collect.sh "폴더"...`** — 지정 폴더 추출 + 영상 일괄 enrich(캡션+STT). 수집의 전 과정.
- `extract_notes.sh ["폴더"]` — 메모 추출만 (collect.sh가 호출하는 하위 도구).
- `enrich_video.sh <URL> [캡션최소길이=50] [언어=ko]` — 영상 1개 캡션+필요시 STT (collect.sh 하위).

## 안전

- 추출은 **읽기**다. 원본 메모를 자동으로 삭제·수정하지 않는다.
- **폴더 선택을 먼저 묻는다** — 잠긴/민감 폴더를 모르고 처리하지 않기 위함이다.
- 링크 해석은 공개 메타데이터·음성 조회 수준 — 인증·로그인이 필요한 콘텐츠를 우회하지 않는다(인스타가 막히면 막힌 대로 표기).
- 어떤 단계에서도 실제 메모 내용을 이 스킬 레포에 쓰지 않는다.
- 해석 결과를 지어내지 않는다 — fetch·STT 실패는 실패로 정직하게 표기한다.
